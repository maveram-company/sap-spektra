Generar un reporte Generate a report

Haz clic en el boton "Generar" de la tarjeta del tipo de reporte que deseas. Click the "Generar" (Generate) button on the report type card you want.
El boton muestra un spinner de carga y el texto cambia a "Generando...". Mientras se genera un reporte, los botones de las demas tarjetas se desactivan para evitar generaciones simultaneas. The button shows a loading spinner and the text changes to "Generando...". While a report is being generated, the buttons on other cards are disabled to prevent simultaneous generations.
Una vez generado, el reporte aparece como una nueva fila en la tabla de reportes generados (debajo de las tarjetas). Ademas, se muestra una notificacion toast en la esquina inferior derecha confirmando la generacion exitosa. Once generated, the report appears as a new row in the generated reports table (below the cards). Additionally, a toast notification appears in the bottom-right corner confirming successful generation.

Notificacion toast Toast notification

Al completar la generacion, aparece una notificacion verde con el mensaje de exito. Ejemplo: When generation completes, a green notification appears with the success message. Example:

✅ Reporte diario generado exitosamente Daily report generated successfully

La notificacion desaparece automaticamente despues de 3 segundos, o puedes cerrarla manualmente haciendo clic en la X que aparece a la derecha del mensaje. The notification automatically disappears after 3 seconds, or you can manually dismiss it by clicking the X that appears to the right of the message.

Columnas de la tabla Table columns

  • Fecha — fecha y hora exactas en que se genero el reporte, en formato local colombiano (sin AM/PM, formato 24h). Fecha (Date) — exact date and time the report was generated, in Colombian locale format (24h, no AM/PM).
  • Tipo — badge con el nombre del tipo de reporte (Diario, Semanal, Salud, Auditoria). Tipo (Type) — badge with the report type name (Diario, Semanal, Salud, Auditoria).
  • Estado — badge verde con un punto indicador y el texto OK, confirmando que el reporte se genero correctamente. Estado (Status) — green badge with a dot indicator and OK text, confirming the report was generated correctly.
  • Accion — boton "Descargar" con un icono de flecha hacia abajo. Al hacer clic, se descarga un archivo JSON con los datos del reporte. Accion (Action) — "Descargar" (Download) button with a down-arrow icon. Clicking it downloads a JSON file with the report data.

Archivo descargado Downloaded file

El archivo descargado es un JSON con el siguiente nombre: RPT-001-diario-2026-03-11.json (el nombre incluye el ID del reporte, el tipo y la fecha). The downloaded file is a JSON with the following name: RPT-001-diario-2026-03-11.json (the name includes the report ID, type and date).

El contenido del JSON incluye un resumen con: The JSON content includes a summary with:

  • id — identificador unico del reporte (p. ej., RPT-001). id — unique report identifier (e.g., RPT-001).
  • type — tipo de reporte (diario, semanal, salud, auditoria). type — report type (diario, semanal, salud, auditoria).
  • generatedAt — marca de tiempo ISO 8601 de cuando se genero. generatedAt — ISO 8601 timestamp of when it was generated.
  • summary — objeto con conteo de sistemas, eventos, alertas (activas y resueltas). summary — object with system count, events, alerts (active and resolved).
  • status — estado del reporte (OK). status — report status (OK).

Ejemplo de contenido JSON JSON content example

  "id": "RPT-001",

  "generatedAt": "2026-03-11T14:32:05.000Z",

    "systemsCount": 9,
