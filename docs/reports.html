Generacion de ReportesReport Generation

Los reportes pueden generarse de forma automatica (programados) o bajo demanda via API o interfaz web. Reports can be generated automatically (scheduled) or on-demand via API or web interface.

Solicitud de ReporteReport Request
El usuario selecciona el tipo de reporte, rango de fechas, sistemas a incluir, y formato de exportacion. Opcionalmente, puede programar la generacion recurrente. The user selects the report type, date range, systems to include, and export format. Optionally, recurring generation can be scheduled.
Recopilacion de DatosData Collection
El backend recopila datos de multiples fuentes: metricas de sistemas, alertas, eventos, runbooks, operaciones, y certificados. Los datos se agregan segun el periodo solicitado. The backend collects data from multiple sources: system metrics, alerts, events, runbooks, operations, and certificates. Data is aggregated according to the requested period.
Procesamiento y FormatoProcessing and Formatting
Los datos se procesan, se calculan estadisticas, se generan graficas, y se formatean en la plantilla del tipo de reporte seleccionado. Data is processed, statistics calculated, charts generated, and formatted in the selected report type template.
DistribucionDistribution
El reporte se almacena internamente y opcionalmente se envia por email, Slack, o Teams a los destinatarios configurados. Se guarda un historial de reportes generados. The report is stored internally and optionally sent via email, Slack, or Teams to configured recipients. A history of generated reports is kept.

Alertas criticasCritical alerts COUNT(alerts) WHERE severity='critical' TodosAll
Tasa de auto-resolucionAuto-resolution rate (autoResolved / total) * 100 Diario, SemanalDaily, Weekly

Eventos por componenteEvents by component GROUP BY component, COUNT(*) Diario, SemanalDaily, Weekly
