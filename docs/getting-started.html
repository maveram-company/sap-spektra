Opcion A: Instalacion con Docker (Recomendada) Option A: Docker Installation (Recommended)

La forma mas rapida de arrancar SAP Spektra es usando Docker Compose, que levanta PostgreSQL, Redis y la API automaticamente. The fastest way to start SAP Spektra is using Docker Compose, which brings up PostgreSQL, Redis and the API automatically.

Clonar el repositorioClone the repository 
git clone https://github.com/maveram-company/sap-spektra.git
cd sap-spektra
Configurar variables de entornoConfigure environment variables 
cp apps/api/.env.example apps/api/.env

El archivo .env.example incluye valores por defecto que funcionan con Docker Compose. The .env.example file includes default values that work with Docker Compose.

Levantar los serviciosStart the services 
docker-compose up -d

Esto inicia PostgreSQL (puerto 5432), Redis (puerto 6379) y la API (puerto 3001). This starts PostgreSQL (port 5432), Redis (port 6379) and the API (port 3001).

Ejecutar migraciones y seedRun migrations and seed 
cd apps/api
npx prisma migrate dev --name init
npm run seed
Iniciar el frontendStart the frontend 
cd frontend
npm install
npm run dev
Abrir en el navegadorOpen in browser

Navega aNavigate to http://localhost:5173

Opcion B: Instalacion sin Docker Option B: Installation without Docker

Iniciar PostgreSQL localmenteStart PostgreSQL locally 
# macOS
brew services start postgresql@16

# Linux (systemd)
sudo systemctl start postgresql
Crear la base de datosCreate the database 
createdb sap_spektra
# o via psql:
psql -c "CREATE DATABASE sap_spektra;"
Iniciar Redis localmenteStart Redis locally 
# macOS
brew services start redis

# Linux (systemd)
sudo systemctl start redis-server
Configurar la URL de la base de datosConfigure the database URL 
# apps/api/.env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/sap_spektra"
REDIS_URL="redis://localhost:6379"
JWT_SECRET="your-super-secret-key-change-in-production"
RUNTIME_MODE="LOCAL_SIMULATED"
Instalar dependencias de la APIInstall API dependencies 
cd apps/api
npm install
Ejecutar migraciones PrismaRun Prisma migrations 
npx prisma generate
npx prisma migrate dev --name init
Poblar datos de demoSeed demo data 
npm run seed
Iniciar la APIStart the API 
npm run start:dev

La API estara disponible en http://localhost:3001 The API will be available at http://localhost:3001

Iniciar el frontend (nueva terminal)Start the frontend (new terminal) 
cd frontend
npm install
npm run dev

El frontend estara disponible en http://localhost:5173 The frontend will be available at http://localhost:5173

Troubleshooting

PostgreSQL no inicia (archivo PID) PostgreSQL won't start (PID file)

❌
Error: FATAL: lock file "postmaster.pid" already exists Error: FATAL: lock file "postmaster.pid" already exists

Solucion: Elimina el archivo PID huerfano y reinicia: Solution: Remove the orphan PID file and restart: 

# macOS
rm /usr/local/var/postgresql@16/postmaster.pid
brew services restart postgresql@16

# Linux
sudo rm /var/lib/postgresql/16/main/postmaster.pid
sudo systemctl restart postgresql

Puerto 3001 o 5173 en uso Port 3001 or 5173 in use

❌
Error: EADDRINUSE: address already in use :::3001 Error: EADDRINUSE: address already in use :::3001

Solucion: Identifica y elimina el proceso que usa el puerto: Solution: Identify and kill the process using the port: 

# Encontrar el proceso
lsof -i :3001
# Terminar el proceso
kill -9 <PID>

Error de permisos en Prisma Prisma permission error

❌
Error: PrismaClientInitializationError: Can't reach database server Error: PrismaClientInitializationError: Can't reach database server

Solucion: Verifica que PostgreSQL esta corriendo y que la URL en .env es correcta. Si usas Docker, asegurate de que el contenedor esta levantado: Solution: Verify that PostgreSQL is running and the URL in .env is correct. If using Docker, ensure the container is up: 

docker-compose ps
docker-compose logs postgres

Redis no se conecta Redis won't connect

Solucion: La aplicacion funciona sin Redis (modo degradado sin cache). Para verificar Redis: Solution: The application works without Redis (degraded mode without cache). To verify Redis: 

redis-cli ping
# Debe responder: PONG

El seed falla o no crea datos Seed fails or creates no data

Solucion: Asegurate de haber ejecutado las migraciones antes del seed: Solution: Make sure you ran migrations before the seed: 

cd apps/api
npx prisma migrate dev
npm run seed
✅
Verificacion: Despues de completar la instalacion, navega a http://localhost:3001/health para verificar que la API esta saludable. Deberias ver {"status":"ok"}. Verification: After completing installation, navigate to http://localhost:3001/health to verify the API is healthy. You should see {"status":"ok"}.
