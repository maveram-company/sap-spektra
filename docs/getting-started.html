Instalacion Paso a PasoStep-by-Step Installation

Paso 1: Infraestructura con DockerStep 1: Infrastructure with Docker

Levanta PostgreSQL 16 y Redis 7 con Docker Compose: Start PostgreSQL 16 and Redis 7 with Docker Compose: 

docker compose up -d

Esto inicia los contenedores de base de datos y cache en segundo plano. This starts the database and cache containers in the background.

✅
Verificacion: Ejecuta docker ps para confirmar que los contenedores postgres y redis estan corriendo. Verification: Run docker ps to confirm the postgres and redis containers are running.

Paso 2: Configurar el BackendStep 2: Configure the Backend

Instalar dependenciasInstall dependencies 
cd backend
npm install
Configurar variables de entornoConfigure environment variables 
cp .env.example .env

El archivo .env.example incluye valores predeterminados para desarrollo local.The .env.example file includes default values for local development.

Ejecutar migraciones de PrismaRun Prisma migrations 
npx prisma migrate dev
Sembrar datos inicialesSeed initial data 
npx prisma db seed
Iniciar el servidorStart the server 
npm run start:dev

Backend disponible en http://localhost:3001Backend available at http://localhost:3001

Paso 3: Configurar el FrontendStep 3: Configure the Frontend

Instalar dependenciasInstall dependencies 
cd frontend
npm install
Iniciar el servidor de desarrolloStart the development server 
npm run dev

Frontend disponible en http://localhost:5173Frontend available at http://localhost:5173

Solucion de ProblemasTroubleshooting

El backend no conecta a la base de datosBackend cannot connect to database

  • Verifica que Docker esta corriendo: docker psVerify Docker is running: docker ps
  • Confirma que DATABASE_URL en .env apunta a localhost:5432Confirm DATABASE_URL in .env points to localhost:5432
  • Reinicia los contenedores: docker compose down && docker compose up -dRestart containers: docker compose down && docker compose up -d

Error de migraciones de PrismaPrisma migration errors

  • Reset completo: npx prisma migrate reset (borra datos existentes)Full reset: npx prisma migrate reset (deletes existing data)
  • Regenerar el cliente: npx prisma generateRegenerate client: npx prisma generate

El frontend no conecta al backendFrontend cannot connect to backend

  • Verifica que el backend esta corriendo en el puerto 3001Verify the backend is running on port 3001
  • Confirma VITE_API_URL=http://localhost:3001 en el .env del frontendConfirm VITE_API_URL=http://localhost:3001 in the frontend .env
  • Asegurate de que VITE_USE_MOCKS=false para usar el backend realMake sure VITE_USE_MOCKS=false to use the real backend
