3. Roles de Usuario 3. User Roles

SAP Spektra utiliza un sistema de 4 roles jerarquicos. Cada rol tiene un nivel numerico que determina sus permisos. Un rol superior hereda todos los permisos de los roles inferiores. SAP Spektra uses a 4-tier hierarchical role system. Each role has a numeric level that determines its permissions. A higher role inherits all permissions from lower roles.

🛡
Admin
Nivel 40 — Acceso total Level 40 — Full access
  • Gestionar usuarios y organizacion Manage users and organization
  • Modificar configuracion y planes Modify settings and plans
  • Acceso a todas las operaciones Access to all operations
  • Aprobar cualquier accion Approve any action
🚀
Escalation
Nivel 30 — Operador avanzado Level 30 — Advanced operator
  • Aprobar operaciones criticas Approve critical operations
  • Modificar reglas de alertas Modify alert rules
  • Ejecutar runbooks avanzados Execute advanced runbooks
  • Gestionar incidentes escalados Manage escalated incidents
🛠
Operator
Nivel 20 — Operador estandar Level 20 — Standard operator
  • Monitorear sistemas en tiempo real Monitor systems in real time
  • Ejecutar runbooks pre-aprobados Execute pre-approved runbooks
  • Gestionar incidentes Manage incidents
  • Crear y responder alertas Create and respond to alerts
👁
Viewer
Nivel 10 — Solo lectura Level 10 — Read only
  • Ver dashboards y metricas View dashboards and metrics
  • Consultar reportes View reports
  • Ver estado de sistemas View system status
  • Sin permisos de ejecucion No execution permissions
⚠️
Importante: La seccion Admin del menu solo es visible para usuarios con rol Admin. Las opciones de Configuracion son accesibles para Admin y Escalation. Important: The Admin menu section is only visible to Admin role users. Settings options are accessible to Admin and Escalation roles.

SAP Landscape

Debajo de los KPIs encontraras la grilla del SAP Landscape: una coleccion de tarjetas que representan cada sistema SAP conectado. Cada tarjeta de sistema muestra: Below the KPIs you will find the SAP Landscape grid: a collection of cards representing each connected SAP system. Each system card shows:

  • SID — Identificador del sistema (ej: EP1, BP1, SOL). SID — System identifier (e.g., EP1, BP1, SOL).
  • Indicador de salud — Gauge circular con porcentaje (verde ≥ 90%, amarillo 70-89%, rojo < 70%). Health gauge — Circular gauge with percentage (green ≥ 90%, yellow 70-89%, red < 70%).
  • Tipo — Tipo de sistema SAP (S/4HANA, BW/4HANA, ECC, etc.). Type — SAP system type (S/4HANA, BW/4HANA, ECC, etc.).
  • Base de datos — Motor de base de datos (HANA, Oracle, SQL Server, etc.). Database — Database engine (HANA, Oracle, SQL Server, etc.).
  • Entorno — DEV, QAS o PRD. Environment — DEV, QAS or PRD.
  • Breaches — Numero de SLA breaches activos en ese sistema. Breaches — Number of active SLA breaches on that system.

If there are operations awaiting approval, the Dashboard shows a prominent alert with the number of pending approvals. Click it to go directly to the approvals queue.
