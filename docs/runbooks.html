RunbooksRunbooks

18 runbooks integrados + ejecucion automatica 18 built-in runbooks + automatic execution

La pantalla de Runbooks le permite explorar y ejecutar procedimientos de automatizacion predefinidos sobre sus sistemas SAP. Cada runbook encapsula una serie de pasos operativos que SAP Spektra ejecuta de forma segura, registrando cada accion y su resultado. La vista se divide en dos pestanas principales: Catalogo y Ejecuciones. The Runbooks screen lets you explore and execute predefined automation procedures on your SAP systems. Each runbook encapsulates a series of operational steps that SAP Spektra executes safely, logging every action and its result. The view is divided into two main tabs: Catalog and Executions.

En la parte superior de la pantalla vera dos pestanas: At the top of the screen you will see two tabs:

Pestana: CatalogoTab: Catalog

Al abrir la pestana Catalogo, vera una tabla con todos los runbooks disponibles. Cada fila representa un runbook con la siguiente informacion: When you open the Catalog tab, you will see a table with all available runbooks. Each row represents a runbook with the following information:

ColumnaColumn Que muestraWhat it shows
ID Identificador unico del runbook en formato RB-001, RB-002, etc. Se muestra en fuente monoespaciada color cian.Unique runbook identifier in format RB-001, RB-002, etc. Displayed in cyan monospace font.
Nombre + DescripcionName + Description Nombre del runbook en negrita (ej. Restart Application Server) seguido de una breve descripcion de lo que hace el procedimiento.Runbook name in bold (e.g. Restart Application Server) followed by a brief description of what the procedure does.
Cost-Safe Indica si el runbook es seguro en terminos de coste e impacto:
• Si — El runbook se considera seguro y puede ejecutarse de inmediato sin aprobacion.
• No — El runbook puede tener impacto en coste o disponibilidad y requiere aprobacion antes de ejecutarse. Indicates whether the runbook is safe in terms of cost and impact:
• Si — The runbook is considered safe and can be executed immediately without approval.
• No — The runbook may impact cost or availability and requires approval before execution.
ModoMode Modo de ejecucion del runbook:
• Auto — Se ejecuta automaticamente cuando una alerta dispara la condicion asociada.
• Aprobacion — Requiere que un usuario con rol de escalamiento apruebe la ejecucion antes de que se inicie. Runbook execution mode:
• Auto — Executes automatically when an alert triggers the associated condition.
• Aprobacion — Requires a user with escalation role to approve execution before it starts.
DB / TipoType Base de datos o tipo de sistema al que aplica el runbook (ej. HANA, ASE, MaxDB, All).Database or system type the runbook applies to (e.g. HANA, ASE, MaxDB, All).
Runs Cantidad total de veces que se ha ejecutado este runbook en el entorno.Total number of times this runbook has been executed in the environment.
Exito %Success % Porcentaje de ejecuciones exitosas sobre el total. Se muestra como barra de progreso o valor numerico (ej. 94.5%).Percentage of successful executions out of total. Shown as a progress bar or numeric value (e.g. 94.5%).
DuracionDuration Duracion promedio de ejecucion del runbook (ej. 2m 30s).Average runbook execution duration (e.g. 2m 30s).
TCode Codigo de transaccion SAP relacionado, mostrado en fuente monoespaciada (ej. SM21, ST22, DB02).Related SAP transaction code, shown in monospace font (e.g. SM21, ST22, DB02).
PrerequisitosPrerequisites Lista de condiciones o permisos necesarios antes de ejecutar el runbook (ej. Acceso SSH, Usuario <SID>adm).List of conditions or permissions required before executing the runbook (e.g. SSH Access, User <SID>adm).
AccionesActions Dos botones disponibles por cada runbook:
• Ejecutar — Lanza la ejecucion real del runbook sobre el sistema seleccionado. Si el runbook es Cost-Safe = No, se redirige al flujo de aprobacion.
• Dry-run — Simula la ejecucion sin aplicar cambios reales. Muestra los pasos que se ejecutarian y su resultado esperado. Two buttons available per runbook:
• Execute — Launches the real runbook execution on the selected system. If the runbook has Cost-Safe = No, it redirects to the approval flow.
• Dry-run — Simulates execution without applying real changes. Shows the steps that would be executed and their expected result.
Regla de SeguridadSafety Rule

Los runbooks marcados como Cost-Safe = Si se ejecutan inmediatamente cuando se disparan por una alerta automatica o cuando usted pulsa Ejecutar. Los runbooks con Cost-Safe = No siempre pasan por el flujo de aprobacion — incluso en modo automatico, la ejecucion se detiene y genera una solicitud de aprobacion que debera gestionar desde la pantalla de Aprobaciones. Runbooks marked as Cost-Safe = Yes execute immediately when triggered by an automatic alert or when you click Execute. Runbooks with Cost-Safe = No always go through the approval flow — even in automatic mode, execution stops and generates an approval request that must be managed from the Approvals screen.

ConsejosTips

  • Utilice Dry-run antes de ejecutar un runbook por primera vez en un sistema de produccion para verificar que los prerequisitos estan satisfechos.Use Dry-run before executing a runbook for the first time on a production system to verify that prerequisites are met.
  • Los runbooks con bajo porcentaje de exito pueden indicar que faltan prerequisitos en ciertos sistemas — revise la columna Prerequisitos para mas detalles.Runbooks with a low success percentage may indicate missing prerequisites on certain systems — check the Prerequisites column for more details.
  • Puede filtrar la tabla de ejecuciones por sistema, runbook o resultado usando los controles de busqueda en la parte superior de la tabla.You can filter the executions table by system, runbook or result using the search controls at the top of the table.
  • Para revisar aprobaciones pendientes de runbooks no cost-safe, visite la pantalla Aprobaciones.To review pending approvals for non-cost-safe runbooks, visit the Approvals screen.
