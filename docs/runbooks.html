● SAFE

Libera memoria no utilizada en instancias SAP HANA ejecutando procedimientos de reclaim en los servicios indexserver y nameserver. Reclaims unused memory on SAP HANA instances by executing reclaim procedures on indexserver and nameserver services.

Validacion de CompatibilidadCompatibility Validation

Antes de ejecutar un runbook, Spektra valida automaticamente que el sistema objetivo cumple con todos los requisitos. La validacion verifica: Before executing a runbook, Spektra automatically validates that the target system meets all requirements. The validation checks:

  • Stack Type: ABAP, JAVA, DUAL_STACK, EDGE — el runbook debe ser compatible con el stack del sistema.Stack Type: ABAP, JAVA, DUAL_STACK, EDGE — the runbook must be compatible with the system's stack.
  • Base de Datos: HANA, Oracle, MSSQL, DB2, ASE, MaxDB — se valida la categoria de BD.Database: HANA, Oracle, MSSQL, DB2, ASE, MaxDB — the DB category is validated.
  • Sistema Operativo: LINUX (SLES, RHEL, Ubuntu), Windows, AIX, HP-UX, Solaris.Operating System: LINUX (SLES, RHEL, Ubuntu), Windows, AIX, HP-UX, Solaris.
  • Pre-requisitos: HSR, Cluster/Pacemaker, HA configurado, perfil de monitoreo FULL_STACK.Prerequisites: HSR, Cluster/Pacemaker, HA configured, FULL_STACK monitoring profile.
  • Capacidad de ejecucion: El sistema debe tener supportsRunbookExecution = true.Execution capability: The system must have supportsRunbookExecution = true.

Simulacion Dry-RunDry-Run Simulation

La simulacion dry-run permite previsualizar el resultado de una ejecucion sin modificar ningun sistema. Retorna el gate asignado, los pasos, la validacion de compatibilidad y la accion que se crearia. Dry-run simulation lets you preview the result of an execution without modifying any system. Returns the assigned gate, steps, compatibility validation, and the action that would be created.
