Validacion de CompatibilidadCompatibility Validation

Antes de ejecutar un runbook, el sistema valida automaticamente que el runbook sea compatible con el sistema objetivo. Se evaluan 4 dimensiones de compatibilidad: Before executing a runbook, the system automatically validates that the runbook is compatible with the target system. 4 compatibility dimensions are evaluated:

💾
Tipo de Base de DatosDatabase Type

Cada runbook especifica las bases de datos compatibles. Ejemplo: un runbook de "HANA Delta Merge" solo es compatible con sistemas HANA. Un runbook de "Oracle Tablespace" solo funciona con Oracle. Each runbook specifies compatible databases. Example: a "HANA Delta Merge" runbook is only compatible with HANA systems. An "Oracle Tablespace" runbook only works with Oracle. 

compatibleDatabases: ["HANA", "HANA_CLOUD"]
⚙
SAP StackSAP Stack

Algunos runbooks son especificos para ciertos productos SAP. Ejemplo: un runbook de "BW Process Chain" solo aplica a BW/4HANA. Un runbook de "SolMan LMDB" solo aplica a Solution Manager. Some runbooks are specific to certain SAP products. Example: a "BW Process Chain" runbook only applies to BW/4HANA. A "SolMan LMDB" runbook only applies to Solution Manager. 

compatibleProducts: ["S4HANA", "ECC", "BW4HANA"]
💻
Sistema OperativoOperating System

Runbooks que ejecutan comandos a nivel de SO son especificos de plataforma. Ejemplo: un runbook que usa systemctl solo funciona en Linux (SLES/RHEL). Uno que usa sapcontrol es cross-platform. Runbooks that execute OS-level commands are platform-specific. Example: a runbook using systemctl only works on Linux (SLES/RHEL). One using sapcontrol is cross-platform. 

compatibleOS: ["SLES", "RHEL"]
🛡
Alta DisponibilidadHigh Availability

Runbooks de HA/DR requieren que el sistema tenga configuracion de alta disponibilidad activa. Se valida el tipo de HA (HANA_HSR, ASCS_ERS) y el estado del cluster. HA/DR runbooks require the system to have active high availability configuration. The HA type (HANA_HSR, ASCS_ERS) and cluster status are validated. 

requiresHA: true
compatibleHATypes: ["HANA_HSR"]

Bloqueo de Sistemas RISERISE System Blocking

⚠
Restriccion RISE: Los runbooks que requieren acceso al sistema operativo (requiresOSAccess: true) estan bloqueados para sistemas con despliegue RISE_MANAGED. El backend rechaza la ejecucion con error 403 RISE_RESTRICTED. Esto incluye: reinicio de servicios, operaciones de filesystem, comandos de kernel, y operaciones de HA/DR a nivel de cluster. RISE Restriction: Runbooks requiring OS access (requiresOSAccess: true) are blocked for systems with RISE_MANAGED deployment. The backend rejects execution with error 403 RISE_RESTRICTED. This includes: service restart, filesystem operations, kernel commands, and cluster-level HA/DR operations.

Motor de EjecucionExecution Engine

🎭
LOCAL_SIMULATED
DEMO

En modo demo, los runbooks se ejecutan localmente con tiempos de espera simulados. Cada paso genera un resultado pre-configurado con retrasos realistas (2-15 segundos por paso). No se ejecutan comandos reales en ningun sistema. In demo mode, runbooks are executed locally with simulated wait times. Each step generates a pre-configured result with realistic delays (2-15 seconds per step). No real commands are executed on any system.

  • Tiempos de ejecucion simuladosSimulated execution times
  • Resultados pre-configurados (success/fail configurable)Pre-configured results (configurable success/fail)
  • Sin impacto en sistemas realesNo impact on real systems
  • Ideal para demos y trainingIdeal for demos and training
🚀
AWS_REAL
PRODUCTION

En modo produccion, los comandos se envian al agente Spektra instalado en el servidor SAP. El agente ejecuta cada paso secuencialmente y reporta resultados en tiempo real al backend. In production mode, commands are sent to the Spektra agent installed on the SAP server. The agent executes each step sequentially and reports results in real time to the backend.

  • Ejecucion real via agenteReal execution via agent

Ejecucion Paso a PasoStep-by-Step Execution

Cada runbook se compone de una serie de pasos (steps) que se ejecutan secuencialmente. El motor de ejecucion rastrea el estado de cada paso individualmente. Each runbook consists of a series of steps that are executed sequentially. The execution engine tracks the status of each step individually.

Ciclo de Vida de un StepStep Lifecycle

PENDING  →  RUNNING  →  SUCCESS
                    ↘  FAILED
                    ↘  SKIPPED (si un gate lo determina)
EstadoStatus DescripcionDescription IconoIcon
PENDING El paso esta en cola, esperando a que el paso anterior finaliceThe step is queued, waiting for the previous step to finish ◯
RUNNING El paso se esta ejecutando activamente en el sistema objetivoThe step is actively executing on the target system ⚡
SUCCESS El paso se completo exitosamente — se procede al siguienteThe step completed successfully — proceed to next ✓
FAILED El paso fallo — la ejecucion se detiene (o se ejecuta rollback si esta configurado)The step failed — execution stops (or rollback executes if configured) ✗
SKIPPED El paso fue omitido por un gate de seguridad o condicion previa no cumplidaThe step was skipped by a safety gate or unmet precondition →

Tipos de GateGate Types

🛡
SAFE Gate
AUTOMATIC

Gates de seguridad automaticos que validan condiciones antes de continuar con el siguiente paso. Si la condicion no se cumple, la ejecucion se pausa o se aborta. Ejemplos: verificar que el backup previo existe, confirmar que no hay usuarios conectados, validar espacio en disco suficiente. Automatic safety gates that validate conditions before continuing to the next step. If the condition is not met, execution pauses or aborts. Examples: verify previous backup exists, confirm no users connected, validate sufficient disk space. 

{
  "type": "SAFE",
  "condition": "disk_free_pct > 20",
  "onFail": "ABORT",
  "message": "Insufficient disk space for operation"
}
✋
HUMAN Gate
MANUAL

Gates que requieren aprobacion humana explicita para continuar. La ejecucion se pausa hasta que un operador con permisos adecuados apruebe la continuacion. Se usa para operaciones de alto riesgo en produccion. Gates that require explicit human approval to continue. Execution pauses until an operator with proper permissions approves continuation. Used for high-risk operations in production. 

{
  "type": "HUMAN",
  "requiredRole": "escalation",
  "timeout": 3600,
  "onTimeout": "ABORT",
  "message": "Approve HANA restart on PRD?"
}

Modo Dry-RunDry-Run Mode

El modo dry-run permite validar un runbook contra un sistema sin ejecutar ningun comando real. Verifica compatibilidad, permisos, pre-condiciones, y genera un plan de ejecucion detallado. Dry-run mode allows validating a runbook against a system without executing any real commands. It verifies compatibility, permissions, preconditions, and generates a detailed execution plan.
