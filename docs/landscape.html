Topologia por SIDTopology by SID

Debajo de las tarjetas de resumen, la pagina presenta una grilla de tarjetas agrupadas por SID. Cada tarjeta representa un sistema SAP y su topologia descubierta. Below the summary cards, the page displays a card grid grouped by SID. Each card represents an SAP system and its discovered topology.

Encabezado de la Tarjeta de Topologia Topology Card Header
  • SID — El identificador del sistema (por ejemplo, S4P, BWP) mostrado de forma prominente. SID — The system identifier (for example, S4P, BWP) displayed prominently.
  • Producto — El tipo de producto SAP del sistema: S/4HANA, BW/4HANA, ECC, Solution Manager, etc. Product — The SAP product type of the system: S/4HANA, BW/4HANA, ECC, Solution Manager, etc.
  • Badge de ambiente — Un badge de color que identifica el entorno del sistema: PRD (rojo para produccion), QAS (ambar para calidad), DEV (cian para desarrollo). Environment badge — A colored badge identifying the system environment: PRD (red for production), QAS (amber for quality), DEV (cyan for development).
Estado de Alta Disponibilidad (HA) High Availability (HA) Status

Cada tarjeta de topologia incluye un badge de escudo que indica la configuracion de alta disponibilidad del sistema: Each topology card includes a shield badge indicating the system's high availability configuration:

  • HSR — HANA System Replication activo. HSR — HANA System Replication active.
  • ASCS_ERS — Cluster ASCS/ERS configurado. ASCS_ERS — ASCS/ERS cluster configured.
  • Standalone — Sin configuracion de alta disponibilidad. Standalone — No high availability configuration.
Lista de NodosNodes List

Dentro de cada tarjeta de topologia veras una lista de los nodos que componen el sistema. Cada nodo muestra: Inside each topology card, you will see a list of the nodes that make up the system. Each node displays:

  • Icono de servidor — Un icono visual que representa el nodo. Server icon — A visual icon representing the node.
  • Nombre del rol — La funcion que cumple el nodo dentro del sistema, por ejemplo: Primary App, HANA Primary, HANA Secondary, ASCS, ERS. Role name — The function the node serves within the system, for example: Primary App, HANA Primary, HANA Secondary, ASCS, ERS.
  • Hostname — El nombre de host del servidor donde corre la instancia, por ejemplo: sapapp01, hanadb01. Hostname — The host name of the server where the instance runs, for example: sapapp01, hanadb01.
💡
La topologia se descubre automaticamente durante el escaneo de conexion. No es necesario definir manualmente los nodos ni roles — SAP Spektra los detecta e identifica por ti. The topology is automatically discovered during the connection scan. You do not need to manually define nodes or roles — SAP Spektra detects and identifies them for you.

Tabla de Instancias DescubiertasDiscovery Instances Table

En la parte inferior de la pagina se encuentra una tabla detallada con todas las instancias descubiertas. La tabla incluye un campo de busqueda en la parte superior que permite filtrar resultados por ID de instancia, hostname, SID, rol o producto. At the bottom of the page, there is a detailed table with all discovered instances. The table includes a search field at the top that allows you to filter results by instance ID, hostname, SID, role, or product.
