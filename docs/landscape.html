Proposito del LandscapeLandscape Purpose

La pagina de Landscape proporciona una vista completa del entorno SAP, validando la salud de cada sistema mediante cuatro verificaciones automaticas. Permite identificar rapidamente problemas de conectividad, configuracion y rendimiento en todo el landscape. The Landscape page provides a complete view of the SAP environment, validating each system's health through four automatic checks. It allows quick identification of connectivity, configuration and performance issues across the entire landscape.

Estado general: Se calcula como el peor resultado entre los cuatro chequeos. Si uno falla, el sistema se marca como Fail. Si al menos uno es advertencia y ninguno falla, el sistema es Warn. Overall status: Calculated as the worst result among the four checks. If one fails, the system is marked as Fail. If at least one is warning and none fail, the system is Warn.

4 Verificaciones por Sistema4 Checks per System

Cada sistema SAP registrado se somete a cuatro verificaciones automaticas que determinan su estado en el landscape: Each registered SAP system undergoes four automatic checks that determine its landscape status:

🔗
1. Conectividad1. Connectivity
Pass

Verifica que el sistema SAP es alcanzable desde la plataforma. Se realiza un ping al conector configurado y se valida que el tiempo de respuesta esta dentro de los umbrales aceptables. Verifies that the SAP system is reachable from the platform. A ping is performed on the configured connector and the response time is validated against acceptable thresholds.

  • Ping RFC/HTTP al sistema destinoRFC/HTTP ping to target system
  • Timeout maximo: 5 segundosMaximum timeout: 5 seconds
  • Reintentos automaticos: 3Automatic retries: 3
📈
2. Monitoreo2. Monitoring
Pass

Confirma que la recoleccion de metricas esta activa y que se estan recibiendo datos recientes. Valida que el ultimo dato recibido no supere el umbral de staleness. Confirms that metrics collection is active and recent data is being received. Validates that the last received data does not exceed the staleness threshold.

  • Ultimo dato recibido < 5 minutosLast data received < 5 minutes
  • Metricas de CPU, memoria, disco activasCPU, memory, disk metrics active
  • Alertas de staleness configurablesConfigurable staleness alerts
🛡
3. Configuracion HA3. HA Configuration
Warn

Verifica que la configuracion de alta disponibilidad es correcta. Incluye validacion de cluster, replicacion de base de datos y configuracion de failover. Verifies that the high availability configuration is correct. Includes cluster validation, database replication and failover configuration.

  • Cluster Pacemaker/Corosync activoActive Pacemaker/Corosync cluster
  • Replicacion HANA System ReplicationHANA System Replication status
  • Failover automatico configuradoAutomatic failover configured
💙
4. Indice de Salud4. Health Score
Pass

Calcula un puntaje compuesto de salud (0-100) basado en multiples metricas: CPU, memoria, disco, tiempos de respuesta, errores y carga de trabajo. Calculates a composite health score (0-100) based on multiple metrics: CPU, memory, disk, response times, errors and workload.

  • Pass: ≥ 80 puntosPass: ≥ 80 points
  • Warn: 50-79 puntosWarn: 50-79 points
  • Fail: < 50 puntosFail: < 50 points
