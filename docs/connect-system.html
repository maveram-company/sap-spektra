Spektra Agent
On-Premise

Agente ligero instalado en el servidor SAP. Comunicacion cifrada via WebSocket con compresion. Lightweight agent installed on the SAP server. Encrypted communication via WebSocket with compression.

  • Instalacion en 5 minutos5-minute installation
  • Bajo consumo de recursos (<50MB RAM)Low resource consumption (<50MB RAM)
  • Conexion saliente (sin abrir puertos entrantes)Outbound connection (no inbound ports)
  • Auto-actualizacion seguraSecure auto-update
  • Soporte Linux/WindowsLinux/Windows support
ⓘ
Ideal para sistemas on-premise sin SAP BTP. Ideal for on-premise systems without SAP BTP.

Legacy

  • Acceso a BAPIs y Function ModulesAccess to BAPIs and Function Modules
  • Soporte SNC para cifradoSNC support for encryption
  • Requiere usuario de servicio SAPRequires SAP service user

Requiere apertura de puertos 33XX y 32XX en el firewall. Requires opening ports 33XX and 32XX in the firewall.

Spektra Agent — Despliegue— Deployment

Instrucciones de DespliegueDeployment Instructions

Descargar el agenteDownload the agent

Descargue la ultima version del agente Spektra para su sistema operativo:Download the latest version of the Spektra agent for your operating system:

# Linux (SLES, RHEL, Ubuntu)
curl -fsSL https://get.spektra.io/agent | sudo bash

# Windows (PowerShell como Admin)
iwr -useb https://get.spektra.io/agent.ps1 | iex
Configurar el token de registroConfigure the registration token

Use el token generado por Spektra para vincular el agente a su tenant:Use the token generated by Spektra to link the agent to your tenant:

# Linux
sudo spektra-agent configure \
  --token "stk_reg_xxxxxxxxxxxxxxxx" \
  --sid PRD \
  --instance-nr 00

# El token expira en 24 horas / The token expires in 24 hours
Iniciar el servicioStart the service 
# Linux (systemd)
sudo systemctl enable spektra-agent
sudo systemctl start spektra-agent

# Verificar status / Verify status
sudo systemctl status spektra-agent
Verificar en SpektraVerify in Spektra

El sistema aparecera automaticamente en la lista de sistemas una vez que el agente se conecte.The system will automatically appear in the system list once the agent connects.

📦
Requisitos del Agente:Agent Requirements: Linux (kernel 4.15+) o Windows Server 2016+, 50MB RAM, 100MB disco, acceso HTTPS saliente al puerto 443. Linux (kernel 4.15+) or Windows Server 2016+, 50MB RAM, 100MB disk, outbound HTTPS access on port 443.

Solucion de ProblemasTroubleshooting

Error: Timeout de conexionError: Connection timeout

Verifique que los puertos estan abiertos en el firewall y que el hostname es accesible desde la red de Spektra.Verify that the ports are open in the firewall and the hostname is reachable from the Spektra network.

# Test de conectividad
telnet sapapp01.prod.local 3200
nc -zv sapapp01.prod.local 3200

Error: Autenticacion fallidaError: Authentication failed

Verifique que el usuario de servicio existe, no esta bloqueado, y tiene los roles correctos asignados.Verify that the service user exists, is not locked, and has the correct roles assigned.

# Verificar en SAP:
# SU01 — Estado del usuario
# SU53 — Verificar autorizaciones faltantes
# SM21 — Log del sistema para errores de login

Error: Agente no se conectaError: Agent not connecting

Verifique que el agente puede realizar conexiones HTTPS salientes al puerto 443.Verify the agent can make outbound HTTPS connections on port 443.

# Verificar logs del agente
sudo journalctl -u spektra-agent -f

# Verificar conectividad saliente
curl -v https://api.spektra.io/health

# Verificar token
sudo spektra-agent status
