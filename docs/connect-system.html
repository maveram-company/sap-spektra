Conectar un Sistema SAPConnect an SAP System

Guia paso a paso para conectar un sistema SAP a Spektra mediante Agente (on-premise) o Cloud Connector (RISE) Step-by-step guide to connect an SAP system to Spektra via Agent (on-premise) or Cloud Connector (RISE)

Metodos de ConexionConnection Methods

🚀
Metodo 1: AgenteMethod 1: Agent
ON_PREMISE

Para sistemas SAP on-premise o en IaaS (AWS, Azure, GCP). El agente Spektra se instala directamente en el servidor y proporciona acceso completo a metricas y operaciones. For on-premise SAP systems or IaaS (AWS, Azure, GCP). The Spektra agent is installed directly on the server and provides full access to metrics and operations.

  • Acceso completo a SO y SAPFull OS and SAP access
  • Todos los runbooks disponiblesAll runbooks available
  • HA/DR operationsHA/DR operations
☁
Metodo 2: Cloud ConnectorMethod 2: Cloud Connector
RISE

Para sistemas gestionados bajo RISE with SAP. Se conecta mediante SAP Cloud Connector (SCC) a traves de un subaccount BTP. Acceso limitado a capa de aplicacion. For systems managed under RISE with SAP. Connects via SAP Cloud Connector (SCC) through a BTP subaccount. Access limited to application layer.

  • Solo metricas SAPSAP metrics only
  • Runbooks limitadosLimited runbooks
  • Sin acceso a SONo OS access

# Download and run installation script

  sudo bash -s -- \
    --api-key "spk_example_xxxxxxxxxxxxxxxxxxxxxxxx" \
    --sid "PRD" \

Paso 3: Verificar conectividadStep 3: Verify connectivity

# Test connectivity to Spektra backend
sudo spektra-agent test-connection

# Expected output:
# ✓ DNS resolution: agent.spektra.maveram.com → 52.xx.xx.xx
# ✓ TLS handshake: OK (TLS 1.3)
# ✓ Authentication: API key valid
# ✓ Heartbeat: sent and acknowledged (latency: 45ms)
# ✓ SAP detection: SID=PRD, Instance=00, DB=HANA
# ✓ All checks passed!

Paso 4: Configurar usuario SAPStep 4: Configure SAP user

# The agent needs an RFC user in the SAP system
# Create user SPEKTRA_MON in SU01 with the following roles:
#   - SAP_BC_MON_ADMIN (Monitoring)
#   - SAP_BC_BAS_MON (Basis Monitoring)
#   - Z_SPEKTRA_AGENT (custom role - see documentation)

# Configure RFC credentials in the agent
sudo spektra-agent configure-sap \
  --client 100 \
  --user SPEKTRA_MON \
  --password "********" \
  --language EN

Configuracion de SAP Cloud Connector (RISE)SAP Cloud Connector Setup (RISE)

Para sistemas RISE with SAP, la conexion se establece mediante SAP Cloud Connector (SCC). Este metodo no requiere instalar software en el servidor SAP. For RISE with SAP systems, the connection is established through SAP Cloud Connector (SCC). This method does not require installing software on the SAP server.

Configurar BTP SubaccountConfigure BTP Subaccount
Acceder a SAP BTP Cockpit y navegar al subaccount donde esta configurado el Cloud Connector. Verificar que el SCC esta conectado (Status: Connected). Access SAP BTP Cockpit and navigate to the subaccount where the Cloud Connector is configured. Verify the SCC is connected (Status: Connected).
Registrar Spektra como DestinoRegister Spektra as Destination
Crear un nuevo destination en BTP pointing a la API de Spektra con las credenciales OAuth2 proporcionadas por Spektra. Create a new destination in BTP pointing to the Spektra API with the OAuth2 credentials provided by Spektra.
Configurar Mapping en SCCConfigure Mapping in SCC
En el Cloud Connector, agregar un mapping para el sistema SAP RISE, exponiendo los recursos necesarios (RFC, HTTP) a traves del tunel seguro. In the Cloud Connector, add a mapping for the RISE SAP system, exposing the necessary resources (RFC, HTTP) through the secure tunnel.

En el wizard de Spektra, ingresar el BTP subaccount ID, el location ID del SCC, y las credenciales de servicio generadas. In the Spektra wizard, enter the BTP subaccount ID, the SCC location ID, and the generated service credentials.

Configuracion del SIDSID Configuration

El SID se configura durante la instalacion del agente o la conexion via Cloud Connector. Asegurate de que el SID coincida exactamente con el SID configurado en el sistema SAP (sapcontrol -function GetSystemInstanceList). The SID is configured during agent installation or Cloud Connector connection. Ensure the SID exactly matches the SID configured in the SAP system (sapcontrol -function GetSystemInstanceList).

⚠
Importante: El SID debe coincidir exactamente con el del sistema SAP (3 caracteres, mayusculas). Un SID incorrecto causara que el agente no pueda detectar los procesos SAP ni recopilar metricas. Ademas, el SID debe ser unico dentro de tu organizacion en Spektra. Important: The SID must exactly match the SAP system's (3 characters, uppercase). An incorrect SID will cause the agent to be unable to detect SAP processes or collect metrics. Also, the SID must be unique within your organization in Spektra.

Configuracion de CredencialesCredential Setup

El agente necesita credenciales para acceder al sistema SAP via RFC. Se recomienda crear un usuario tecnico dedicado con las autorizaciones minimas necesarias. The agent needs credentials to access the SAP system via RFC. It is recommended to create a dedicated technical user with the minimum necessary authorizations.

Language EN Idioma de logon (EN recomendado para consistencia)Logon language (EN recommended for consistency)

ℹ
Seguridad: Las credenciales se almacenan cifradas con AES-256 en el agente local y nunca se transmiten en texto plano al backend de Spektra. Solo se usa la conexion RFC local entre el agente y el sistema SAP. Security: Credentials are stored encrypted with AES-256 on the local agent and are never transmitted in plain text to the Spektra backend. Only the local RFC connection between the agent and the SAP system is used.
