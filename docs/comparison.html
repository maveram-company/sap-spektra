Descripcion GeneralOverview

La funcion de comparacion de sistemas permite seleccionar entre 2 y 6 sistemas SAP para visualizar sus metricas lado a lado. Es especialmente util para: The system comparison feature allows selecting between 2 and 6 SAP systems to view their metrics side by side. It is especially useful for:

  • Comparar PRD vs QAS vs DEV: Verificar que los sistemas de un mismo track tengan configuraciones consistentesCompare PRD vs QAS vs DEV: Verify that systems in the same track have consistent configurations

  • Planificacion de capacidad: Comparar utilizacion de recursos entre sistemas similaresCapacity planning: Compare resource utilization across similar systems
  • Validacion post-migracion: Verificar que un sistema migrado funciona igual que el originalPost-migration validation: Verify that a migrated system performs the same as the original

Validacion de Linea de SIDsSID Line Validation

Al seleccionar sistemas para comparacion, Spektra valida que los SIDs sean validos (3 caracteres, formato correcto) y que pertenezcan al tenant del usuario. La seleccion se puede hacer por SID o por UUID. When selecting systems for comparison, Spektra validates that the SIDs are valid (3 characters, correct format) and belong to the user's tenant. Selection can be done by SID or UUID. 

// SID validation in comparison
const validateComparisonInput = (sids: string[]) => {
  if (sids.length < 2) throw new Error('Minimum 2 systems required');
  if (sids.length > 6) throw new Error('Maximum 6 systems allowed');

  const SID_REGEX = /^[A-Z][A-Z0-9]{2}$/;
  for (const sid of sids) {
    if (!SID_REGEX.test(sid)) {
      throw new Error(`Invalid SID format: ${sid}`);
    }
  }

  // Check for duplicates
  const unique = new Set(sids);
  if (unique.size !== sids.length) {
    throw new Error('Duplicate SIDs not allowed');
  }

  return true;
};
⚠
Limite: Se permite un maximo de 6 sistemas por comparacion para mantener la legibilidad de la vista y el rendimiento de las consultas. Limit: A maximum of 6 systems per comparison is allowed to maintain view readability and query performance.

Comparacion de Health ScoreHealth Score Comparison

La comparacion de health score muestra la evolucion historica del puntaje de salud de cada sistema seleccionado en el mismo grafico temporal, permitiendo identificar correlaciones y divergencias. The health score comparison shows the historical evolution of each selected system's health score on the same time chart, allowing identification of correlations and divergences.

Seleccion Multi-SistemaMulti-System Selection

La interfaz de seleccion permite elegir sistemas de multiples formas: The selection interface allows choosing systems in multiple ways:

Por SID IndividualBy Individual SID

Seleccionar sistemas uno por uno desde un dropdown que muestra SID, producto, entorno, y health score actual. Select systems one by one from a dropdown showing SID, product, environment, and current health score.

Por Filtro RapidoBy Quick Filter

Filtros predefinidos: "Todos los PRD", "Mismo producto", "Mismo track (PRD/QAS/DEV)", "Misma BD", "Mismo OS". Predefined filters: "All PRD", "Same product", "Same track (PRD/QAS/DEV)", "Same DB", "Same OS".

ℹ
Consejo: Para comparar el track completo de un producto, usa el filtro "Mismo track" que selecciona automaticamente PRD, QAS, DEV y SBX del mismo producto SAP. Tip: To compare the full track of a product, use the "Same track" filter which automatically selects PRD, QAS, DEV, and SBX of the same SAP product.
