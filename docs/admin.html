Activacion de Health CheckHealth Check Triggering

Los administradores pueden forzar un health check inmediato en uno o multiples sistemas. Esto es util cuando se sospecha que las metricas automaticas no reflejan el estado real (ej: despues de un mantenimiento de red). Administrators can force an immediate health check on one or multiple systems. This is useful when automatic metrics are suspected not to reflect the real state (e.g., after network maintenance).

Health Check IndividualIndividual Health Check

POST /api/systems/:id/health-check
Authorization: Bearer <admin_jwt>

// Response
{
  "systemId": "a1b2c3d4-...",

  "previousScore": 96,
  "newScore": 94,
  "checkDuration": "2.3s",
  "metricsUpdated": {
    "cpu": 38.2,
    "memory": 72.1,
    "disk": 52.8
  },
  "timestamp": "2025-01-15T14:30:00Z"
}

Eliminacion de SistemasSystem Deletion

⚠
Operacion Destructiva: Eliminar un sistema borra permanentemente todos los datos asociados: metricas historicas, alertas, eventos, ejecuciones de runbooks, jobs, transportes, y certificados. Esta accion es irreversible y requiere confirmacion explicita. Destructive Operation: Deleting a system permanently removes all associated data: historical metrics, alerts, events, runbook executions, jobs, transports, and certificates. This action is irreversible and requires explicit confirmation.

Proceso de EliminacionDeletion Process

Solicitud: El admin envia un DELETE request con el motivo de la eliminacion y la confirmacion del SID. Request: The admin sends a DELETE request with the deletion reason and SID confirmation.
Validacion: El backend verifica que el sistema no tiene operaciones en curso, runbooks en ejecucion, ni aprobaciones pendientes. Validation: The backend verifies the system has no in-progress operations, running runbooks, or pending approvals.
Cascade Delete: Se eliminan en cascada: alertas, eventos, metricas, ejecuciones, jobs, transportes, certificados, y configuraciones HA/DR. Cascade Delete: Cascade deletes: alerts, events, metrics, executions, jobs, transports, certificates, and HA/DR configurations.
Audit: Se registra un evento de seguridad con quien elimino, cuando, que sistema, y el motivo proporcionado. Audit: A security event is logged with who deleted, when, which system, and the provided reason. 
DELETE /api/systems/:id
Authorization: Bearer <admin_jwt>
Content-Type: application/json

{
  "confirmSID": "ECP",
  "reason": "System decommissioned — migrated to S/4HANA (PRD)"
}

// Response
{
  "deleted": true,
  "sid": "ECP",
  "cascadeDeleted": {
    "alerts": 42,
    "events": 1847,
    "runbookExecutions": 23,
    "jobs": 156,
    "transports": 89,
    "certificates": 4
  }
}
