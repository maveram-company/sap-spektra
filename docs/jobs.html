Descripcion GeneralOverview

Spektra proporciona una vista centralizada de todos los jobs de background de SAP (equivalente a la transaccion SM37) en todos los sistemas del landscape. Esto elimina la necesidad de conectarse a cada sistema individualmente para monitorear jobs criticos. Spektra provides a centralized view of all SAP background jobs (equivalent to the SM37 transaction) across all landscape systems. This eliminates the need to connect to each system individually to monitor critical jobs.

ℹ
Fuente de datos: Los datos de jobs se recopilan via el agente Spektra que ejecuta RFC_READ_TABLE sobre la tabla TBTCO (job overview) y TBTCP (job steps) en cada sistema SAP. La recoleccion se realiza cada 60 segundos. Data source: Job data is collected via the Spektra agent executing RFC_READ_TABLE on the TBTCO table (job overview) and TBTCP (job steps) in each SAP system. Collection happens every 60 seconds.

Clases de JobJob Classes

SAP clasifica los jobs en tres clases de prioridad que determinan el orden de ejecucion cuando los work processes de background (BTC) estan limitados. SAP classifies jobs into three priority classes that determine execution order when background work processes (BTC) are limited.

A
Clase A — CriticaClass A — Critical

Maxima prioridad. Se ejecuta en el primer work process disponible, desplazando jobs de menor prioridad si es necesario. Usar solo para jobs esenciales de negocio. Highest priority. Runs on the first available work process, displacing lower-priority jobs if necessary. Use only for business-essential jobs.

Ejemplos: RDDIMPDP (transport import), RSBTCDEL (job log cleanup), BW process chains criticas Examples: RDDIMPDP (transport import), RSBTCDEL (job log cleanup), critical BW process chains

B
Clase B — NormalClass B — Normal

Prioridad estandar para la mayoria de jobs planificados. Se ejecuta cuando hay work processes disponibles despues de los jobs clase A. Standard priority for most scheduled jobs. Runs when work processes are available after class A jobs.

Ejemplos: RSPO0041 (spool cleanup), informes financieros nocturnos, extractores BW Examples: RSPO0041 (spool cleanup), nightly financial reports, BW extractors

C
Clase C — BajaClass C — Low

Prioridad mas baja. Se ejecuta solo cuando hay recursos libres. Ideal para jobs de housekeeping y reportes no urgentes. Lowest priority. Runs only when there are free resources. Ideal for housekeeping jobs and non-urgent reports.

Ejemplos: RSBDCREO (batch input reorganization), reportes estadisticos, archivado de datos Examples: RSBDCREO (batch input reorganization), statistical reports, data archiving

Detalle del JobJob Details

interface SapJob {
  id:           string;     // Internal ID
  jobName:      string;     // Job name (e.g., "RSBTCDEL")
  jobId:        string;     // SAP job number
  systemId:     string;     // System UUID
  sid:          string;     // SID (e.g., "PRD")
  client:       string;     // SAP client (e.g., "100")
  user:         string;     // Job creator/owner (e.g., "BATCHUSER")
  status:       JobStatus;  // running | scheduled | finished | failed | canceled
  jobClass:     JobClass;   // A | B | C
  startDate:    ISO8601;    // Actual or planned start
  endDate:      ISO8601;    // Actual end (null if running/scheduled)
  duration:     number;     // Duration in seconds (null if not finished)
  programName:  string;     // ABAP program (e.g., "RSBTCDEL")
  variant:      string;     // Variant name
  targetServer: string;     // Application server (e.g., "sapapp01_PRD_00")
  spoolList:    string[];   // Associated spool request IDs
  hasLog:       boolean;    // Whether job log is available
  steps:        JobStep[];  // Step details (ABAP report, external command, etc.)
  periodic:     boolean;    // Whether this is a periodic (recurring) job
  errorMessage: string;     // Error message if failed
}

Seguimiento de Cliente y UsuarioClient and User Tracking

Spektra rastrea el cliente SAP y el usuario propietario de cada job para auditorias y analisis. Esto permite identificar rapidamente quien programo un job y en que mandante se ejecuta. Spektra tracks the SAP client and owner user of each job for audits and analysis. This allows quickly identifying who scheduled a job and on which client it runs.

Seguimiento de DuracionDuration Tracking

Spektra rastrea la duracion de cada job y la compara con un baseline historico para detectar anomalias. Un job que toma significativamente mas tiempo que su promedio historico genera una alerta de tipo warning. Spektra tracks each job's duration and compares it with a historical baseline to detect anomalies. A job taking significantly longer than its historical average generates a warning alert.
