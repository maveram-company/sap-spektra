Metodos de ConexionConnection Methods

Spektra ofrece cuatro metodos de conexion para comunicarse con los sistemas SAP. La eleccion depende del modelo de despliegue y los requisitos de seguridad de la organizacion. Spektra offers four connection methods to communicate with SAP systems. The choice depends on the deployment model and the organization's security requirements.

🚀
Spektra Agent
ON_PREMISE

Agente ligero instalado directamente en el servidor SAP. Proporciona el nivel mas alto de visibilidad y control operativo. Se comunica con el backend de Spektra via HTTPS sobre puerto 443. Lightweight agent installed directly on the SAP server. Provides the highest level of visibility and operational control. Communicates with the Spektra backend via HTTPS on port 443.

CaracteristicasFeatures

  • Recoleccion de metricas OS (CPU, memoria, disco, red)OS metrics collection (CPU, memory, disk, network)
  • Ejecucion de runbooks a nivel de SO y SAPOS-level and SAP-level runbook execution
  • Monitoreo de procesos SAP (disp+work, ICM, etc.)SAP process monitoring (disp+work, ICM, etc.)
  • Recoleccion de logs del sistemaSystem log collection
  • Actualizacion remota automaticaAutomatic remote updates
# Agent communication flow
Agent (on-prem) --HTTPS/443--> Spektra API Gateway --> Backend
                <-- Heartbeat Response (30s interval) --
☁
SAP Cloud Connector
RISE

Integracion via SAP Cloud Connector (SCC) para sistemas gestionados bajo RISE with SAP. El SCC establece un tunel seguro entre el subaccount BTP y el sistema SAP. Integration via SAP Cloud Connector (SCC) for systems managed under RISE with SAP. The SCC establishes a secure tunnel between the BTP subaccount and the SAP system.

CaracteristicasFeatures

  • Conexion segura via BTP subaccountSecure connection via BTP subaccount
  • Metricas SAP a nivel de aplicacionSAP application-level metrics
  • Sin acceso a nivel de SO (restriccion RISE)No OS-level access (RISE restriction)
  • Runbooks limitados a capa de aplicacionRunbooks limited to application layer
⚠
Restriccion: Los conectores Cloud Connector no permiten ejecutar runbooks que requieran acceso al sistema operativo, reiniciar servicios, ni realizar operaciones de HA/DR. Restriction: Cloud Connector connectors do not allow executing runbooks that require OS access, service restart, or HA/DR operations.
🔗
RFC/BAPI
DIRECT

Conexion directa al sistema SAP mediante llamadas RFC (Remote Function Call) y BAPIs. Utiliza el protocolo nativo de SAP para ejecutar funciones remotas. Requiere que las credenciales RFC esten configuradas en Spektra. Direct connection to the SAP system via RFC (Remote Function Call) and BAPI calls. Uses SAP's native protocol for remote function execution. Requires RFC credentials to be configured in Spektra.

  • Acceso nativo a funciones SAPNative access to SAP functions
  • Lectura de tablas y ejecucion de transaccionesTable reading and transaction execution
  • Requiere usuario RFC con autorizaciones adecuadasRequires RFC user with proper authorizations
  • Puerto 33XX (donde XX = instance number)Port 33XX (where XX = instance number)
🌐
API Gateway
HYBRID

Conexion mediante un API Gateway intermedio (por ejemplo, SAP API Management en BTP). Util para landscapes complejos donde se requiere un punto centralizado de entrada con politicas de seguridad adicionales. Connection through an intermediate API Gateway (e.g., SAP API Management on BTP). Useful for complex landscapes requiring a centralized entry point with additional security policies.

  • Proxy centralizado para multiples sistemasCentralized proxy for multiple systems
  • Politicas de rate limiting y throttlingRate limiting and throttling policies
  • Transformacion de protocolo (OData, REST)Protocol transformation (OData, REST)
  • Logging y auditing centralizadoCentralized logging and auditing

Mecanismo de HeartbeatHeartbeat Mechanism

El heartbeat es el mecanismo principal para determinar la conectividad y salud del conector. Cada conector envia un paquete de heartbeat al backend de Spektra a intervalos regulares. The heartbeat is the primary mechanism for determining connector connectivity and health. Each connector sends a heartbeat packet to the Spektra backend at regular intervals.

Envio de HeartbeatHeartbeat Send
El agente/conector envia un POST a /api/connectors/heartbeat cada 30 segundos con metricas basicas del sistema (CPU, memoria, disco, timestamp). The agent/connector sends a POST to /api/connectors/heartbeat every 30 seconds with basic system metrics (CPU, memory, disk, timestamp).
Procesamiento en BackendBackend Processing
El backend actualiza lastHeartbeat, recalcula el connectorStatus y actualiza las metricas del sistema. Si hay cambios significativos, se generan eventos. The backend updates lastHeartbeat, recalculates the connectorStatus, and updates system metrics. If there are significant changes, events are generated.
Respuesta con ComandosResponse with Commands
La respuesta del heartbeat puede incluir comandos pendientes para el agente (ejecutar runbook, recopilar logs, actualizar configuracion). The heartbeat response can include pending commands for the agent (execute runbook, collect logs, update configuration).
Deteccion de DesconexionDisconnection Detection
Un job programado revisa cada 60 segundos si algun conector ha excedido el umbral de 3 minutos sin heartbeat. Si es asi, se marca como disconnected y se genera una alerta. A scheduled job checks every 60 seconds if any connector has exceeded the 3-minute heartbeat threshold. If so, it is marked as disconnected and an alert is generated.

Monitoreo de LatenciaLatency Monitoring

Spektra mide la latencia de cada conector en tiempo real. La latencia se calcula como el tiempo de ida y vuelta (RTT) del heartbeat, medido desde el envio hasta la recepcion de la respuesta. Spektra measures each connector's latency in real time. Latency is calculated as the round-trip time (RTT) of the heartbeat, measured from send to response receipt.

Instalacion del Agente (Resumen)Agent Installation (Overview)

El agente Spektra se instala con un solo comando. El script de instalacion detecta automaticamente el sistema operativo, configura el servicio y establece la primera conexion con el backend. The Spektra agent is installed with a single command. The installation script automatically detects the operating system, configures the service, and establishes the first connection to the backend.

Linux (SLES / RHEL)

curl -fsSL https://agent.spektra.maveram.com/install.sh | \
  sudo bash -s -- \
    --api-key "sk_live_xxxxxxxxxxxxxxxx" \
    --sid "PRD" \
    --region "eu-central-1"

Windows (PowerShell)

Invoke-WebRequest -Uri "https://agent.spektra.maveram.com/install.ps1" `
  -OutFile "$env:TEMP\install.ps1"
& "$env:TEMP\install.ps1" `
  -ApiKey "sk_live_xxxxxxxxxxxxxxxx" `
  -SID "PRD" `
  -Region "eu-central-1"
