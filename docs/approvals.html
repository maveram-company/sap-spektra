AprobacionesApprovals

Gestiona las solicitudes de ejecucion de runbooks Manage runbook execution requests

La pantalla de Aprobaciones centraliza todas las solicitudes de ejecucion de runbooks que requieren intervencion humana. Cuando un runbook marcado como Cost-Safe = No se dispara — ya sea por una alerta automatica o por accion manual —, se genera una solicitud de aprobacion que aparece aqui. Solo los usuarios con rol de escalamiento pueden aprobar o rechazar solicitudes. The Approvals screen centralizes all runbook execution requests that require human intervention. When a runbook marked as Cost-Safe = No is triggered — either by an automatic alert or manual action —, an approval request is generated and appears here. Only users with the escalation role can approve or reject requests.

Acciones en Solicitudes PendientesActions on Pending Requests

En la pestana Pendientes, los usuarios con rol de escalamiento veran dos botones de accion junto a cada solicitud: In the Pending tab, users with the escalation role will see two action buttons next to each request:

AprobarApprove RechazarReject
  • Aprobar — Autoriza la ejecucion del runbook. El sistema procede a ejecutar el runbook de inmediato sobre el sistema indicado. La solicitud pasa al estado APROBADA y se registra la ejecucion en la pantalla de Runbooks > Ejecuciones.Approve — Authorizes the runbook execution. The system proceeds to execute the runbook immediately on the indicated system. The request moves to APPROVED status and execution is logged in the Runbooks > Executions screen.
  • Rechazar — Deniega la ejecucion. El runbook no se ejecutara. La solicitud pasa al estado RECHAZADA. Se le solicitara un motivo opcional de rechazo.Reject — Denies execution. The runbook will not run. The request moves to REJECTED status. You will be prompted for an optional rejection reason.
Nota sobre PermisosNote on Permissions

Los usuarios sin rol de escalamiento pueden ver las solicitudes de aprobacion pero no veran los botones de Aprobar/Rechazar. Para solicitar acceso a este rol, contacte al administrador desde la seccion Gestion. Users without the escalation role can view approval requests but will not see the Approve/Reject buttons. To request access to this role, contact the administrator from the Management section.

Expiracion de SolicitudesRequest Expiration

Las solicitudes de aprobacion tienen un tiempo limite configurable. Si ninguna accion se toma dentro del periodo establecido, la solicitud pasa automaticamente al estado EXPIRADA y el runbook no se ejecuta. Puede ver estas solicitudes en la pestana Expiradas. El tiempo de expiracion predeterminado se configura desde Configuracion. Approval requests have a configurable time limit. If no action is taken within the established period, the request automatically moves to EXPIRED status and the runbook is not executed. You can view these requests in the Expired tab. The default expiration time is configured from Settings.

ConsejosTips

  • Revise las solicitudes Pendientes regularmente, especialmente las de severidad CRITICAL y HIGH, para evitar que expiren sin accion.Review Pending requests regularly, especially those with CRITICAL and HIGH severity, to prevent them from expiring without action.
  • Las solicitudes de la pestana Aprobadas incluyen un enlace directo al log de ejecucion del runbook resultante.Requests in the Approved tab include a direct link to the resulting runbook execution log.
  • Use la columna Metrica + Valor para evaluar la urgencia real de la solicitud antes de aprobar o rechazar.Use the Metric + Value column to evaluate the real urgency of the request before approving or rejecting.
