Flujo de Aprobaciones Approval Workflow

Gestion de solicitudes de aprobacion para operaciones criticas en sistemas SAP. Cada solicitud esta protegida por un token unico y requiere revision por usuarios autorizados. Management of approval requests for critical operations on SAP systems. Each request is protected by a unique token and requires review by authorized users.

Evidencia de AprobacionApproval Evidence

Cada solicitud de aprobacion puede contener un objeto JSON de evidencia que documenta el contexto de la solicitud: metricas del sistema, estado del landscape, y justificacion. Each approval request can contain a JSON evidence object documenting the request context: system metrics, landscape status, and justification.

Aprobacion via TokenToken-Based Approval

Cada solicitud genera un token UUID unico almacenado en el campo token. Este token permite aprobar o rechazar la solicitud via API sin requerir sesion activa, util para integraciones con sistemas de ticketing o notificaciones por email. Each request generates a unique UUID token stored in the token field. This token allows approving or rejecting the request via API without requiring an active session, useful for integrations with ticketing systems or email notifications.

🔒
Separacion de responsabilidades: Un operador solicita la ejecucion del runbook, y un usuario de escalacion o administrador aprueba o rechaza. Un operador no puede aprobar sus propias solicitudes. Separation of duties: An operator requests the runbook execution, and an escalation or admin user approves or rejects. An operator cannot approve their own requests.
