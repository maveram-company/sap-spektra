Configuracion de NotificacionesNotification Settings

✉
Email

Notificaciones por email via SMTP o AWS SES. Se pueden configurar destinatarios diferentes por tipo de alerta y sistema. Email notifications via SMTP or AWS SES. Different recipients can be configured per alert type and system. 

{
  "email": {
    "enabled": true,
    "provider": "SES",
    "from": "alerts@spektra.com",
    "recipients": {
      "critical": ["oncall@co.com"],
      "warning": ["team@co.com"],
      "info": ["log@co.com"]
    }
  }
}

Integracion con Slack via webhooks. Alertas se envian a canales especificos con formato enriquecido (bloques, botones de accion). Slack integration via webhooks. Alerts are sent to specific channels with rich formatting (blocks, action buttons). 

{
  "slack": {
    "enabled": true,
    "webhookUrl": "https://hooks...",
    "channels": {
      "critical": "#sap-alerts",

      "approvals": "#sap-approvals"

  }
}

Politica de EscalamientoEscalation Policy

Define como escalan las alertas cuando no se atienden dentro del tiempo estipulado. Defines how alerts escalate when not addressed within the stipulated time.

Nivel 1 — Notificacion Inicial (0 min):Level 1 — Initial Notification (0 min):
Se notifica al equipo de operaciones por email + Slack. Destinatario: equipo on-call del sistema afectado. Operations team is notified via email + Slack. Recipient: on-call team for the affected system.
Nivel 2 — Escalamiento (15 min):Level 2 — Escalation (15 min):
Si no hay respuesta, se escala al team lead y se envia notificacion push adicional. Se crea un incidente en el sistema de tickets. If no response, escalate to team lead and send additional push notification. An incident is created in the ticketing system.
Nivel 3 — Gestion (30 min):Level 3 — Management (30 min):
Se notifica al manager de SAP Basis y al CTO. Se incluye un resumen ejecutivo del impacto de negocio estimado. SAP Basis manager and CTO are notified. An executive summary of estimated business impact is included.

Ventanas de MantenimientoMaintenance Windows

Las ventanas de mantenimiento suprimen alertas y excluyen tiempo de downtime del calculo de SLA. Se pueden configurar como recurrentes o puntuales. Maintenance windows suppress alerts and exclude downtime from SLA calculations. They can be configured as recurring or one-time. 

// Maintenance window configuration
{
  "maintenanceWindows": [
    {
      "name": "Weekly Sunday Maintenance",
      "systems": ["PRD", "QAS", "ECP"],
      "schedule": "0 2 * * 0",  // Sundays at 2:00 AM
      "duration": 240,           // 4 hours
      "suppressAlerts": true,
      "excludeFromSLA": true
    },
    {
      "name": "Monthly Patch Window",
      "systems": ["ALL"],
      "schedule": "0 22 1 * *",  // 1st of month at 10 PM
      "duration": 480,            // 8 hours
      "suppressAlerts": true,
      "excludeFromSLA": true
    }
  ]
}

Gestion de API KeysAPI Key Management

Las API keys permiten autenticacion programatica para integraciones externas, agentes, y automatizaciones. Los admins pueden crear y revocar keys. API keys allow programmatic authentication for external integrations, agents, and automations. Admins can create and revoke keys.

AccionAction Endpoint DescripcionDescription
Crear POST /api/settings/api-keys Generar nueva API key con nombre y permisos. El secret se muestra solo una vez.Generate new API key with name and permissions. The secret is shown only once.
Listar GET /api/settings/api-keys Ver todas las API keys (sin mostrar secret). Muestra fecha de creacion y ultimo uso.View all API keys (without showing secret). Shows creation date and last use.
Revocar DELETE /api/settings/api-keys/:id Revocar una API key inmediatamente. Todas las peticiones futuras con esa key seran rechazadas.Revoke an API key immediately. All future requests with that key will be rejected. 
// Create API key
POST /api/settings/api-keys
{
  "name": "Spektra Agent PRD",
  "permissions": ["heartbeat", "metrics", "commands"],
  "expiresIn": "365d"
}

// Response (shown ONCE)
{
  "id": "key-001",
  "name": "Spektra Agent PRD",
  "key": "spk_example_xxxxxxxxxxxxxxxxxxxxxxxxx",
  "prefix": "spk_xxxx...xxxx",
  "createdAt": "2025-01-15T10:00:00Z",
  "expiresAt": "2026-01-15T10:00:00Z"
}
⚠
Importante: El secret completo de la API key solo se muestra una vez al momento de la creacion. Almacenalo de forma segura. Si se pierde, deberas revocar la key y crear una nueva. Important: The full API key secret is only shown once at creation time. Store it securely. If lost, you must revoke the key and create a new one.
