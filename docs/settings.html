3. Roles y Permisos3. Roles & Permissions

SAP Spektra utiliza un modelo jerarquico de roles. Cada rol tiene un nivel numerico y hereda todos los permisos de los roles inferiores. Se presentan como 4 tarjetas organizadas de mayor a menor nivel: SAP Spektra uses a hierarchical role model. Each role has a numeric level and inherits all permissions from lower roles. They are presented as 4 cards organized from highest to lowest level:

Admin
Level 40

Acceso completo a la plataforma. Puede gestionar usuarios, configuracion, facturacion y eliminar recursos. Full platform access. Can manage users, settings, billing, and delete resources.

Gestionar usuariosManage users FacturacionBilling Eliminar sistemasDelete systems Configuracion globalGlobal settings API KeysAPI Keys AuditoriaAudit log Todos los permisos inferioresAll lower permissions
Escalation
Level 30

Puede aprobar operaciones criticas, modificar reglas de alerta y gestionar politicas de escalacion. Can approve critical operations, modify alert rules, and manage escalation policies.

Aprobar operacionesApprove operations Modificar alertasModify alerts Politicas de escalacionEscalation policies Gestionar integracionesManage integrations Todos los permisos inferioresAll lower permissions
Operator
Level 20

Puede ejecutar runbooks, gestionar incidentes, conectar sistemas y operar transportes. Can execute runbooks, manage incidents, connect systems, and operate transports.

Ejecutar runbooksExecute runbooks Gestionar incidentesManage incidents Conectar sistemasConnect systems Health checksHealth checks TransportesTransports Todos los permisos inferioresAll lower permissions
Viewer

Acceso de solo lectura. Puede ver dashboards, metricas, alertas y reportes, pero no puede ejecutar acciones. Read-only access. Can view dashboards, metrics, alerts, and reports, but cannot execute actions.

Ver dashboardView dashboard Ver metricasView metrics Ver alertasView alerts Ver reportesView reports Analisis IA (lectura)AI Analysis (read)
💡
Modelo jerarquico: Cada nivel hereda automaticamente todos los permisos de los niveles inferiores. Por ejemplo, un usuario Escalation (L30) tiene todos los permisos de Operator (L20) y Viewer (L10), ademas de los suyos propios. Hierarchical model: Each level automatically inherits all permissions from lower levels. For example, an Escalation (L30) user has all permissions of Operator (L20) and Viewer (L10), plus their own.
