Orquestacion de FailoverFailover Orchestration

Spektra orquesta el proceso de failover mediante una secuencia de pasos controlados. Cada paso se ejecuta secuencialmente con validaciones entre pasos (gates). Spektra orchestrates the failover process through a sequence of controlled steps. Each step is executed sequentially with between-step validations (gates).

Metricas RPO/RTORPO/RTO Metrics

💾
RPO

Recovery Point Objective — La cantidad maxima aceptable de datos que se pueden perder en caso de desastre. Se mide como la diferencia entre el ultimo dato comprometido y el punto de falla. Recovery Point Objective — The maximum acceptable amount of data that can be lost in case of disaster. Measured as the difference between the last committed data and the failure point. 

// RPO calculation (real-time)
RPO = currentTime - lastReplicatedTimestamp
// For SYNC mode: RPO should always be 0
// For ASYNC mode: RPO = replication lag
🕒
RTO

Recovery Time Objective — El tiempo maximo aceptable para restaurar el servicio despues de un desastre. Incluye tiempo de deteccion, failover, y validacion. Recovery Time Objective — The maximum acceptable time to restore service after a disaster. Includes detection, failover, and validation time. 

// RTO components
RTO = detectionTime + approvalTime +
      failoverTime + validationTime
// HOT_STANDBY typical: 10s + 0s + 15s + 5s = 30s
// WARM_STANDBY typical: 60s + 300s + 600s + 120s = ~18min

Gestion de Nodos Primario/SecundarioPrimary/Secondary Node Management

interface HAConfiguration {
  systemId:       string;
  strategy:       HAStrategy;        // HOT_STANDBY | WARM_STANDBY | etc.
  haType:         HAType;            // HANA_HSR | ASCS_ERS
  primaryNode: {
    hostname:     string;            // e.g., "hana-prd-01"
    ip:           string;            // e.g., "10.0.1.10"
    role:         "PRIMARY";
    status:       "active" | "failed";
    lastSync:     ISO8601;
  };
  secondaryNode: {
    hostname:     string;            // e.g., "hana-prd-02"
    ip:           string;            // e.g., "10.0.2.10"
    role:         "SECONDARY";
    status:       "synced" | "syncing" | "disconnected";
    replicationMode: "SYNC" | "SYNCMEM" | "ASYNC";
    replicationLag:  number;         // Seconds behind primary
  };
  virtualIP:      string;            // Floating VIP
  rpo:            number;            // Current RPO in seconds
  rto:            number;            // Measured RTO in seconds
  lastFailover:   ISO8601 | null;
  lastDRDrill:    ISO8601 | null;
  autoFailover:   boolean;           // Auto failover enabled
}

After each failover or DR drill, Spektra automatically generates an evidence pack documenting each step of the process for audits and post-incident analysis.
