Estrategias de Alta DisponibilidadHigh Availability Strategies

SAP Spektra soporta cinco estrategias de HA/DR, cada una con diferentes compromisos entre costo, RPO y RTO. SAP Spektra supports five HA/DR strategies, each with different trade-offs between cost, RPO, and RTO.

🔥
HOT_STANDBY
RPO: 0 min

Replica sincrona en tiempo real. El nodo secundario esta activo y recibe datos continuamente. Failover automatico en menos de 5 minutos.Synchronous real-time replication. Secondary node is active and receives data continuously. Automatic failover in under 5 minutes.

  • Replicacion sincrona de datosSynchronous data replication
  • Failover automaticoAutomatic failover
  • RTO: ~5 minutosminutes
  • Costo: Alto (infraestructura duplicada)Cost: High (duplicated infrastructure)
🔄
WARM_STANDBY
RPO: 5 min

Replica asincrona con nodo secundario pre-configurado. Requiere arranque manual o semi-automatico del nodo secundario.Asynchronous replication with pre-configured secondary node. Requires manual or semi-automatic secondary node startup.

  • Replicacion asincronaAsynchronous replication
  • Nodo secundario en espera tibiaSecondary node in warm standby
  • RTO: ~15 minutosminutes
  • Costo: MedioCost: Medium
💡
PILOT_LIGHT
RPO: 15 min

Infraestructura minima en el sitio DR. Solo los servicios criticos estan replicados. Se escala al activarse.Minimal infrastructure at the DR site. Only critical services are replicated. Scales up when activated.

  • Infraestructura minima activaMinimal active infrastructure
  • Escalado bajo demandaOn-demand scaling
  • RTO: ~30 minutosminutes
  • Costo: Medio-bajoCost: Medium-low
💾
BACKUP_RESTORE
RPO: 60 min

Restauracion desde backups periodicos. Sin infraestructura activa en el sitio DR. Proceso manual completo.Restoration from periodic backups. No active infrastructure at the DR site. Fully manual process.

  • Backups programadosScheduled backups
  • Restauracion manualManual restoration
  • RTO: ~120 minutosminutes
  • Costo: BajoCost: Low
🌎
CROSS_REGION_DR
Enterprise

Despliegue multi-region con failover automatico entre centros de datos geograficamente distribuidos. Requiere plan Enterprise.Multi-region deployment with automatic failover between geographically distributed data centers. Requires Enterprise plan.

  • Multi-region activo/pasivo o activo/activoMulti-region active/passive or active/active
  • DNS failover automaticoAutomatic DNS failover
  • RPO configurable: 0-15 minutosConfigurable RPO: 0-15 minutes
  • Solo plan EnterpriseEnterprise plan only

Configuracion HA por SistemaHA Configuration per System

ⓘ
La configuracion HA se define por sistema. Cada sistema puede tener una estrategia diferente segun sus requerimientos de negocio. HA configuration is defined per system. Each system can have a different strategy based on its business requirements.

  "systemId": "uuid",

Ejecutar FailoverTrigger Failover

⚠️
Accion Critica:Critical Action: Solo los usuarios con rol admin (nivel 40) pueden ejecutar un failover. Esta accion redirigira el trafico del nodo primario al secundario. Only users with admin role (level 40) can trigger a failover. This action will redirect traffic from the primary to the secondary node.
