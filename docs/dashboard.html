↻
Estrategia de RefrescoRefresh Strategy
  • Auto-refresh: Los KPIs se actualizan automaticamente cada 30 segundos usando polling.Auto-refresh: KPIs update automatically every 30 seconds using polling.
  • Manual: El boton de refrescar en la cabecera recarga todos los datos inmediatamente.Manual: The refresh button in the header reloads all data immediately.
  • Cache: Las respuestas del backend se cachean en Redis con TTL configurable para reducir la carga en PostgreSQL.Cache: Backend responses are cached in Redis with configurable TTL to reduce PostgreSQL load.
  • Optimistic UI: Las acciones de usuario (como aprobar una solicitud) actualizan la UI inmediatamente mientras la peticion se procesa en segundo plano.Optimistic UI: User actions (like approving a request) update the UI immediately while the request is processed in the background.
💡
Multi-tenant: El dashboard solo muestra datos de la organizacion del usuario autenticado. El filtrado por orgId se aplica automaticamente en el backend mediante guards de NestJS. Multi-tenant: The dashboard only shows data for the authenticated user's organization. Filtering by orgId is applied automatically in the backend via NestJS guards.
