EventosEvents

Registro completo de actividad de todos los sistemas y la plataforma. Complete activity log across all systems and the platform.

Vista general de la paginaPage overview

La pagina Eventos (Monitor > Eventos) actua como el registro centralizado de toda la actividad capturada por Spektra. Aqui se recopilan eventos provenientes tanto de los sistemas SAP conectados como de la propia plataforma. Al ingresar, usted ve el encabezado con el titulo "Eventos" y la descripcion "Registro completo de actividad". The Events page (Monitor > Events) acts as the centralized log of all activity captured by Spektra. It gathers events from both connected SAP systems and the platform itself. Upon entering, you see the header with the title "Events" and the description "Complete activity log."

PaginacionPagination

La tabla carga 50 eventos por pagina. Al final de la lista visible, aparece un boton "Cargar mas" que indica cuantos eventos adicionales quedan por mostrar: The table loads 50 events per page. At the bottom of the visible list, a "Load more" button appears showing how many additional events remain:

Al hacer clic en "Cargar mas", se anaden los siguientes 50 eventos al final de la tabla sin perder los que ya estan visibles. El boton actualiza el conteo de eventos restantes cada vez. Cuando no quedan mas eventos por cargar, el boton desaparece. Clicking "Load more" appends the next 50 events to the bottom of the table without losing those already visible. The button updates the remaining count each time. When no more events remain, the button disappears.

💡
La carga por lotes de 50 eventos esta disenada para mantener la pagina rapida y receptiva, incluso cuando el registro total contiene miles de eventos. Los filtros se aplican sobre el conjunto completo de datos, no solo sobre los eventos visibles. Loading events in batches of 50 is designed to keep the page fast and responsive, even when the total log contains thousands of events. Filters apply across the full data set, not just the visible events.

Consejos de usoUsage tips

  • Investigacion de incidentes: Combine el filtro de nivel Critico con un sistema especifico para reconstruir la cronologia de un incidente. Incident investigation: Combine the Critical level filter with a specific system to reconstruct the chronology of an incident.
  • Auditoria de cambios: Filtre por origen Plataforma para ver todos los cambios de configuracion, ejecuciones de runbooks y acciones de usuario realizadas en Spektra. Change audit: Filter by source Platform to see all configuration changes, runbook executions, and user actions performed in Spektra.
  • Busqueda por componente: Si sospecha de un problema en un area especifica (por ejemplo Database), escriba el nombre del componente en el campo de busqueda para aislar rapidamente los eventos relevantes. Component search: If you suspect an issue in a specific area (e.g., Database), type the component name in the search field to quickly isolate relevant events.
  • Correlacion entre sistemas: Seleccione Sistema: Todos y nivel Critico para ver si multiples sistemas estan experimentando problemas simultaneos, lo cual podria indicar un problema de infraestructura compartida. Cross-system correlation: Select System: All and level Critical to see if multiple systems are experiencing simultaneous issues, which could indicate a shared infrastructure problem.
⚠️
Los eventos se retienen segun la politica de retencion configurada en Configuracion. Si necesita acceder a eventos mas antiguos, consulte la seccion de Reportes donde puede generar exportaciones historicas. Events are retained according to the retention policy configured in Settings. If you need to access older events, check the Reports section where you can generate historical exports.
