Agent / Collector

El agente (Python) se despliega junto a los sistemas SAP y recopila metricas del host. Incluye stubs de colectores para multiples bases de datos. El agente usa el endpoint POST /metrics/ingest para enviar datos y PATCH /connectors/:id/heartbeat para heartbeats. The agent (Python) is deployed alongside SAP systems and collects host metrics. It includes collector stubs for multiple databases. The agent uses the POST /metrics/ingest endpoint to send data and PATCH /connectors/:id/heartbeat for heartbeats.

Multi-tenancy

SAP Spektra implementa multi-tenancy a nivel de base de datos usando organizationId como filtro en cada consulta. Cada usuario pertenece a una organizacion via Membership. SAP Spektra implements multi-tenancy at the database level using organizationId as a filter in every query. Each user belongs to an organization via Membership. 

// TenantGuard extrae organizationId del JWT
@UseGuards(JwtAuthGuard, TenantGuard, RolesGuard)

// En cada servicio, las queries filtran por orgId
findAll(orgId: string) {
  return this.prisma.system.findMany({
    where: { organizationId: orgId }
  });
}
⚠️
Aislamiento total: Un usuario de la organizacion A nunca puede ver datos de la organizacion B. El TenantGuard valida que el organizationId del JWT coincida con la membresia del usuario. Total isolation: A user from organization A can never see data from organization B. The TenantGuard validates that the JWT's organizationId matches the user's membership.
