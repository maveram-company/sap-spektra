Formato SIDSID Format

Cada sistema SAP se identifica mediante un SID (System Identifier) de exactamente 3 caracteres alfanumericos. El SID es unico dentro del landscape y sigue las convenciones estandar de SAP. Each SAP system is identified by a SID (System Identifier) of exactly 3 alphanumeric characters. The SID is unique within the landscape and follows standard SAP naming conventions.

ℹ
Regla SAP: El SID debe tener exactamente 3 caracteres. Solo letras mayusculas y digitos. No puede comenzar con un digito. Ejemplos validos: PRD, QAS, S4H, ECP. Ejemplos invalidos: 1AB, PR, PROD. SAP Rule: The SID must be exactly 3 characters. Only uppercase letters and digits. Cannot start with a digit. Valid examples: PRD, QAS, S4H, ECP. Invalid examples: 1AB, PR, PROD. 
// SID validation regex
const SID_REGEX = /^[A-Z][A-Z0-9]{2}$/;

// Examples
"PRD"  // valid - S/4HANA Production
"QAS"  // valid - Quality Assurance
"S4H"  // valid - S/4HANA system
"1AB"  // invalid - starts with digit
"PR"   // invalid - only 2 characters

Modelos de DespliegueDeployment Models

🏢
ON_PREMISE

El sistema SAP se ejecuta en infraestructura propia del cliente (on-premise o IaaS). El agente Spektra se instala directamente en el servidor y tiene acceso completo a las operaciones del sistema. The SAP system runs on the customer's own infrastructure (on-premise or IaaS). The Spektra agent is installed directly on the server and has full access to system operations.

  • Acceso completo a runbooksFull access to runbooks
  • Operaciones sin restriccionUnrestricted operations
  • Monitoreo a nivel de SOOS-level monitoring
  • HA/DR con control totalHA/DR with full control
☁
RISE_MANAGED

El sistema SAP esta gestionado por SAP a traves de RISE with SAP. La conectividad se establece mediante SAP Cloud Connector. Algunas operaciones estan restringidas. The SAP system is managed by SAP through RISE with SAP. Connectivity is established through SAP Cloud Connector. Some operations are restricted.

⚠
Restricciones RISE: Los sistemas RISE no permiten ejecucion directa de runbooks a nivel de SO, reinicio de servicios, ni operaciones HA/DR manuales. Solo se permite monitoreo y runbooks a nivel de aplicacion SAP. RISE Restrictions: RISE systems do not allow direct OS-level runbook execution, service restart, or manual HA/DR operations. Only monitoring and SAP application-level runbooks are permitted.
  • Runbooks limitados (solo nivel aplicacion)Limited runbooks (application level only)
  • Conexion via SAP Cloud ConnectorConnection via SAP Cloud Connector
  • Sin acceso a nivel de SONo OS-level access
  • Monitoreo de metricas SAP unicamenteSAP metrics monitoring only

Campos de SystemMetaSystemMeta Fields

Cada sistema almacena un objeto SystemMeta con toda la informacion de configuracion y estado. Each system stores a SystemMeta object with all configuration and status information. 

interface SystemMeta {
  id:              string;       // UUID v4
  sid:             string;       // 3-char SID (e.g., "PRD")
  type:            SapProduct;   // S4HANA | ECC | BW4HANA | SOLMAN | PO | GRC | CRM | EWM
  environment:     Environment;  // PRD | QAS | DEV | SBX | DR
  host:            string;       // Hostname or IP
  status:          SystemStatus; // healthy | warning | critical | offline
  healthScore:     number;       // 0-100
  cpu:             number;       // CPU % (0-100)
  memory:          number;       // Memory % (0-100)
  disk:            number;       // Disk % (0-100)
  database:        DatabaseType; // HANA | ORACLE | MSSQL | DB2 | ASE | MAXDB | HANA_CLOUD
  databaseVersion: string;       // e.g., "2.0 SPS 07"
  os:              OsType;       // SLES | RHEL | WINDOWS | AIX | SOLARIS
  osVersion:       string;       // e.g., "15 SP4"
  deployment:      Deployment;   // ON_PREMISE | RISE_MANAGED
  region:          string;       // e.g., "us-east-1", "eu-central-1"
  instanceNumber:  string;       // SAP instance number (00-99)
  client:          string;       // SAP client (e.g., "100")
  sapVersion:      string;       // e.g., "2023 FPS02"
  kernelVersion:   string;       // e.g., "793"
  lastHeartbeat:   ISO8601;      // Last agent/connector heartbeat
  connectorType:   ConnectorType;// AGENT | CLOUD_CONNECTOR | RFC | API_GATEWAY
  connectorStatus: ConnStatus;   // connected | degraded | disconnected
  tags:            string[];     // Custom tags
  organizationId:  string;       // Tenant isolation
  createdAt:       ISO8601;
  updatedAt:       ISO8601;
}
